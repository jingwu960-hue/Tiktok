# 计划：自动学习功能实现

## 概述

实现 `tiktok-shop-coach` 技能的自动学习功能，支持智能遍历链接、自动分类存储、双向同步更新。

---

## 当前状态分析

### 已有的功能定义
- SKILL.md 第1054-1082行已定义"链接学习"功能规范
- 第1083-1112行定义了"链接内容自动读取功能"
- 第877-1221行定义了"本地知识库系统"结构

### 缺失的功能
1. **自动抓取机制**：无法自动获取网页内容并遍历链接
2. **智能分类存储**：没有将内容自动分类到知识库文件
3. **双向同步**：没有将新知识同步到SKILL.md的对应章节
4. **触发机制**：仅定义了命令格式，缺少实际触发逻辑

### 可用工具
- **WebFetch**：可抓取网页内容
- **Read/Write/Edit**：文件读写操作
- **MCP Playwright**：可选，用于处理需要交互的页面

---

## 实现方案

### 方案概述

实现一个完整的自动学习流程：
1. **触发** → 命令 `/学习 [URL]` 或自动识别URL
2. **抓取** → WebFetch 获取主页面内容
3. **分析** → 识别页面中的相关链接
4. **遍历** → 智能跳转到相关链接（最多10个）
5. **存储** → 内容分类存储到知识库 + 更新SKILL.md

```markdown
## 自动学习流程

### 功能概述
当用户发送TikTok Shop相关链接时，技能会自动：
1. 获取当前日期（date命令）
2. 抓取链接内容（WebFetch）
3. 提取关键信息（AI分析）
4. 识别相关链接（智能遍历，最多10个）
5. 自动分类存储（知识库 + SKILL.md）

### 智能遍历策略
- 优先级1：同一域名下的课程/文章链接
- 优先级2：同类别话题的相关链接
- 优先级3：工具下载、文档下载链接
- 跳过：登录页、广告页、评论区

### 知识更新规则
1. **知识库优先**：新知识首先存储到 `TikTokShop/知识库/[分类].md`
2. **SKILL.md同步**：关键信息更新到 SKILL.md 的对应章节
3. **索引更新**：自动更新 `index.json` 索引文件
4. **去重处理**：相同内容不重复存储
```
