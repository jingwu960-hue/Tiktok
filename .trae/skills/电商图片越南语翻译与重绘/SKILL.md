# 电商图片越南语翻译与重绘

## 描述
该技能用于处理电商类图片（详情页、海报、活动图等），自动识别图片中的中文文本，将其精准翻译为越南语，并在不改变原设计结构的前提下完成图片重绘，实现可直接用于跨境电商平台的成品图输出。

## 使用场景
当用户上传包含中文的电商图片时使用，例如：
- TikTok Shop 商品详情页
- 电商促销海报
- 产品宣传图
- 商品功能说明图
- 活动 Banner 图
- 商品规格 / 参数表
- 长图详情（多屏滚动图）

目标是：中文 → 越南语本地化 + 保持原排版结构 + 输出可直接商用图片

---

## 指令

### 一、图像解析
1. 识别图片中所有文本内容，包括但不限于：
   - 标题 / 副标题
   - 标签 / 卖点
   - 按钮文案
   - 价格 / 活动信息
   - 说明文字 / 注释
   - 参数表中的中文标签（数字与单位保留）
2. 精确定位每一段文字在图片中的：
   - 位置
   - 层级关系
   - 排版结构（对齐方式、大小、分区、颜色）
   - 旋转角度（倾斜 / 弧形 / 垂直文本）
3. 必须完整识别所有文本，不得遗漏任何区域文字。

### 图像预检（特殊场景前置步骤）
- **低质量图片自动增强**：若图片分辨率低于 720p 或文字模糊，先进行超分辨率 / 锐化处理后再识别。
- **长图切分**：若图片高度大于 4000px，自动按屏切分为多段子图处理，再合并输出。
- **多区域冲突检测**：对密集文字区域进行分段与重排，避免重绘后文字重叠。
- **复杂背景检测**：对渐变、图案、纹理、人物背景上的文字采用"轮廓采样 + 局部 inpaint"方式去除原文字。

---

### 二、翻译处理
1. 将所有中文内容翻译为越南语。
2. 要求：
   - 使用地道越南语表达，而非直译
   - 保持电商营销风格（有吸引力、转化导向）
   - **保留品牌名、型号、专业术语**（或行业通用译法）
   - 确保语义准确且符合当地用户阅读习惯
   - 价格、参数、百分比等数字与单位一律保留（如 "299元" → "299đ"，"50%" → "50%"）
   - 对 emoji 与常见符号保留原样

### 翻译辅助能力（可调用 MCP 工具提升质量）
当用户同时提供商品链接（如 TikTok Shop / Shopee / Lazada 链接）时，允许调用以下工具来获取上下文，以提升翻译准确度：

| MCP 工具 | 作用 | 使用时机 |
|----------|------|----------|
| `playwright_navigate` + `playwright_screenshot` / `browser_take_screenshot` | 打开商品页，获取商品标题、描述、规格参数等原始文案 | 用户提供商品链接时 |
| `playwright_get_visible_text` | 提取商品页可见文本，作为翻译参考上下文 | 需要对照越南语/英文原标题以做术语一致 |
| `playwright_post` / `playwright_get` | （如允许）调用外部翻译 API 或字典，补全专业术语 | 遇到生僻行业词时 |

> 说明：这些工具仅用于"辅助获取商品上下文信息，核对品牌名/型号/参数"，**不得将用户未授权的任何图片上传至外部服务**。

### 上下文翻译提示（在调用大模型翻译时注入 system prompt）
```
你是一个专业的跨境电商本地化翻译。请把中文翻译成地道越南语，满足以下规则：
1. 品牌名 / 型号 / SKU / 数字 / 百分比 / 价格单位（$、¥、đ）一律保留；
2. 营销词使用越南电商常见表达，如 "限时特惠"→"Giảm giá thời hạn"、"热销"→"Bán chạy"、"新品"→"Hàng mới"；
3. 参数表中的属性名翻译为越南语，但数值与单位原样保留；
4. emoji 与符号原样保留；
5. 仅输出翻译后的文本，不要加解释或引号。
```

---

### 三、图片重构
1. 在不改变原设计风格的前提下：
   - 将所有中文替换为对应越南语
   - 保持原版式结构、布局、视觉层级
2. 排版要求：
   - 不允许文字重叠、溢出或错位
   - 根据越南语长度自动调整字号或换行
   - 尽量保持与原设计一致的视觉平衡
   - **保留原文字颜色与背景**（非黑即白判定，需要时采样原文字颜色）
   - 倾斜文字按原角度重绘；垂直文本按原方向重绘
   - 圆角按钮 / 徽章 / 标签类元素使用圆角遮罩绘制
3. 输出必须达到"可直接用于电商投放"的设计质量。

### 复杂排版场景处理（重要）
| 场景 | 处理策略 |
|------|----------|
| 文字叠加在人物/产品图上 | 使用 inpaint + 局部轮廓采样背景色，避免把人物涂抹糊 |
| 文字在渐变/图案背景上 | 采用"透明底板 + 原文字颜色"方式重写，而非填实色块 |
| 价格 / 折扣 / 促销角标 | 保留原形状与颜色，只替换数字单位（¥→đ、元→đ） |
| 参数表 / 规格表 | 仅翻译左侧属性列，右侧数值与单位保留；按原表格对齐 |
| 多行长标题 | 自动按语义换行，保持每行视觉平衡；必要时缩小字号 |
| 垂直/倾斜文字 | 按原角度旋转绘制；角度 > 45° 时使用独立图层叠加 |
| emoji 与特殊符号 | 保留；若目标字体不支持，回退为文字描述或移除 |
| 水印 / Logo | 保留不动，不作为翻译目标（用面积占比与高置信度识别） |

---

### 四、输出要求
1. 仅输出最终结果图片（已替换为越南语版本）。
2. 图片要求：
   - 高清清晰（输出分辨率 ≥ 输入分辨率，不做放大插值）
   - 无水印（翻译过程产生的内部标记全部擦除）
   - 无额外标注说明
   - JPEG 质量 ≥ 95；PNG 透明通道保持
3. 禁止输出：
   - 翻译过程
   - 文本列表
   - 解释说明
4. **质量自检**：输出前自动运行一次质量检查，若以下任一失败则重新处理：
   - OCR 漏检率：核心区域文字漏检 > 1 处
   - 翻译覆盖度：未被替换的中文残留在图片中
   - 视觉一致性：平均颜色偏移 > 阈值，或出现大块误涂抹
5. **批量输出**：当 source 目录存在多张图片时，全部处理后统一输出到 output 目录，文件名保持原 base + `_translated` 后缀。

## 可调用工具（特殊场景下允许使用）

| 能力 | 工具来源 | 用途 |
|------|----------|------|
| 商品页上下文获取 | MCP: `playwright_navigate`, `playwright_screenshot`, `playwright_get_visible_text` | 获取真实商品页标题/规格，保证翻译术语一致 |
| 真实商品页面校验 | MCP: `browser_navigate`, `browser_take_screenshot` | 对照原页面确认输出图片在越南本地化语境中的可读性 |
| 本地脚本执行 | Python: `translate_image.py` | 主工作流：OCR → 翻译 → 重绘 → 输出 |
| 本地质量自检 | Python: `image_precheck.py` / `diagnose.py` | 预检图片质量、检测漏检中文与视觉异常 |
| 对比检查 | Python: `compare_images.py` | 原图与输出图的视觉一致性比对（调试用） |

### 工具调用约束
- 所有图片仅在本地 `TranslationImg/source` / `output` 目录处理；
- 不得上传原始图片到任何第三方服务（仅允许调用本地 OCR 模型与文本翻译 API，文本 API 仅传递已识别文字内容）；
- MCP 浏览器工具仅用于访问公开的商品页面以获取文本上下文。

## 关键约束（必须严格遵守）
- 不得遗漏任何文字内容
- 不得错译或自由改写原意
- 必须逐元素对应替换
- 必须保持原始视觉结构一致
- 必须保证商业可用级别输出质量（可直接用于 TikTok Shop 投放）
- 品牌名 / 型号 / SKU / 数字 / 单位 / 百分比必须原样保留
- 价格中的货币符号必须本地化（¥/元 → ₫）
- emoji 与常见符号（★、✓、→ 等）保留原样
