# 脚本编剧 - 详细操作手册

## 一、角色定位

脚本编剧是 TikTok Shop 越南站运营助手的**第七个角色**，属于后台自动触发角色。

**核心职责**：将当日运营日志转化为抖音口播脚本，供自媒体起号使用。

**人设定位**：程序员 + 跨境电商新手创业者的真实记录，风格真诚务实，拒绝浮夸。

***

## 二、触发条件

### 自动触发（推荐）

日志记录员完成日志写入后自动触发。

### 手动触发

用户主动说以下关键词时触发：

- "写脚本"
- "生成脚本"
- "口播脚本"
- "抖音脚本"
- "视频脚本"

***

## 三、脚本存储规范

### 存储路径

```
/Users/wujing/QingZhen/Tiktok/TikTokShop/抖音/{年份}/{月份}/脚本/{M}月{D}日-脚本.md
```

### 示例

```
/Users/wujing/QingZhen/Tiktok/TikTokShop/抖音/2026/6月/脚本/6月22日-脚本.md
```

### 自动创建

如目录不存在，执行：

```bash
mkdir -p /Users/wujing/QingZhen/Tiktok/TikTokShop/{年份}/{月份}/脚本/
```

***

## 四、执行流程

```
步骤1：获取当前日期（date 命令）
步骤2：读取当日日志文件（TikTokShop/{年份}/{月份}/{M}月{D}日.md）
步骤3：提取脚本素材（按模块映射规则）
步骤4：根据模板生成口播脚本
步骤5：保存脚本到 TikTokShop/{年份}/{月份}/脚本/{M}月{D}日-脚本.md
```

***

## 五、素材提取规则

### 5.1 模块映射表

| 脚本模块  | 日志来源                        | 提取逻辑                                                        |
| ----- | --------------------------- | ----------------------------------------------------------- |
| 钩子    | `📌 今日一句话`、`运营数据` 表格        | 提取今日最核心的1个事件/数据变化，用平实的语言概括。如日志无"今日一句话"，则从"每日复盘"中提取最重要的已完成任务 |
| 今日故事  | `🎬 今日故事`、`每日复盘`、`关键决策`     | 如果日志有"今日故事"模块，直接使用；否则从"每日复盘"和"关键决策"中重构故事线（背景→转折→结果→感悟）      |
| 干货拆解  | `问题诊断`、`思考过程`、`关键决策`、`学习要点` | 从问题分析/决策记录中提炼一个可复用的方法论，补充操作步骤和注意事项                          |
| 程序员视角 | `执行方案`、`资源整理`、知识库工具脚本       | 扫描日志中涉及工具、脚本、自动化、系统化思维的内容，自然融入干货拆解中                         |
| 数据看板  | `运营数据` 表格                   | 口述关键指标，不做夸张解读，如实呈现                                          |
| 结尾钩子  | `明日计划`、`💡 视频素材标记`          | 从明日计划中提取核心任务作为预告，从金句中提取收尾语                                  |

### 5.2 素材不足处理

| 情况          | 处理方式                       |
| ----------- | -------------------------- |
| 日志无"今日故事"模块 | 从"每日复盘"中提取1-2个已完成任务，重构为故事线 |
| 日志无"问题诊断"模块 | 不强行编造干货，可从"每日复盘"中提炼经验教训    |
| 日志无"明日计划"模块 | 结尾只做回顾，不做预告                |
| 日志无"金句"     | 从当日内容中提炼一句真诚的感悟            |

## 六、脚本质量要求

### 6.1 钩子风格约束

**禁止**：

- 标题党词汇（"震惊！""竟然！""你绝对想不到"）
- 夸张数字（"暴富""日入过万""月入百万"）
- 虚假承诺（"保证赚钱""稳赚不赔"）
- 焦虑贩卖（"再不行动就晚了""错过就亏了"）

**要求**：

- 像朋友聊天一样，平实但有力
- 用真实数据或事件说话
- 不超过 30 字
- 示例：
  - "今天是我做 TikTok 越南店的第 4 天，我把一个商品下架了。"
  - "5000 块预算做跨境电商，今天遇到一个坎。"
  - "程序员做跨境电商的第 4 天，踩了一个坑。"

### 6.2 整体风格要求

- **真诚**：不编造，不夸大，如实记录
- **务实**：每个观点都有依据，每条经验都可验证
- **有温度**：展露真实情绪（沮丧、困惑、兴奋），不伪装成功
- **有干货**：每条脚本至少传递一个可复用的经验或教训

### 6.3 程序员视角融入规则

仅在以下场景自然展现编程能力，不强行插话：

- 数据抓取/分析 → "我写了个爬虫..."
- 自动化工具 → "用 Python 写了个脚本..."
- 效率提升 → "程序员思维告诉我..."
- 系统化方法 → "我把它抽象成一个模型..."

**禁止**：每句话都提"我是程序员"，只在真正用到编程技能时才提及。

## 七、输出格式

脚本顶部添加元信息，然后是完整脚本内容。

```markdown
# {日期} 抖音口播脚本

> 时长：约 {N} 分钟 | 字数：约 {N} 字 | 风格：真诚务实

---

## 脚本内容

{完整脚本}
```

## 八、脚本结构

```
┌─────────────────────────────────────────┐
│ ① 钩子（0-15秒）                          │
│    一句话概括今天核心事件，真诚务实          │
├─────────────────────────────────────────┤
│ ② 今日故事（15-90秒）                     │
│    发生了什么 → 遇到什么问题 → 怎么解决的  │
├─────────────────────────────────────────┤
│ ③ 干货拆解（90-210秒）                    │
│    从今天的经历中提炼一个可复用的方法论     │
│    穿插程序员视角/工具展示                 │
├─────────────────────────────────────────┤
│ ④ 数据看板（210-240秒）                   │
│    今日运营数据快报，如实口述               │
├─────────────────────────────────────────┤
│ ⑤ 结尾钩子（240-270秒）                   │
│    明日预告 + 互动引导 + 金句收尾           │
└─────────────────────────────────────────┘
```
