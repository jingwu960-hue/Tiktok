# 每日运营指导工作流程

## 触发条件

用户询问"今天该做什么"、"每日任务"、"第X天"等每日运营指导相关问题时触发。

## 执行步骤

### 1. 获取当前日期
```bash
date "+%Y-%m-%d"
```

### 2. 读取店铺数据
读取 `.trae/data/shop-metrics.json` 获取当前店铺状态。

### 3. 识别冷启动阶段
根据 `day_number` 判断当前阶段：
- **第1-15天**：冷启动期 - 打底测款
- **第16-30天**：成长期 - 投流实测
- **第31-60天**：稳定期 - 稳定出单

### 4. 生成任务清单
根据当前阶段，从任务库中选择最重要的3个任务。

#### 冷启动期任务库（第1-15天）
- 优化商品主图（白底图、场景图）
- 发布短视频（日更3-5条）
- 联系达人合作（寄样100份）
- 优化商品标题和详情页
- 分析竞品数据

#### 成长期任务库（第16-30天）
- 开启商品卡投流（$25/天）
- 短视频投流测试（$25/天）
- 回收达人视频（5-10条）
- 分析投流数据，优化素材
- 筛选高效素材

#### 稳定期任务库（第31-60天）
- 放大高效素材投流
- 聚焦直播带货
- 优化供应链和履约
- 提升店铺评分
- 扩大商品池

### 5. 输出任务清单
使用 `config/templates/daily-tasks.md` 模板生成报告。

### 6. 等待用户反馈
收集用户执行结果，记录到 `shop-metrics.json`。

### 7. 更新数据
根据用户反馈更新 `shop-metrics.json` 中的当日数据。

## 错误处理

### 场景1：数据文件不存在
**处理方式**：
1. 创建默认结构的 `shop-metrics.json`
2. 记录警告日志
3. 使用默认值（day_number=1）继续执行

### 场景2：day_number 缺失
**处理方式**：
1. 根据 start_date 和当前日期计算 day_number
2. 更新到数据文件中

### 场景3：用户未反馈执行结果
**处理方式**：
1. 提醒用户记录执行情况
2. 不强制更新数据
3. 下次对话时再次询问

## 输出格式

使用 `config/templates/daily-tasks.md` 模板输出。

## 示例

**用户输入**："今天该做什么？"

**执行流程**：
1. 获取日期：2026-06-22
2. 读取数据：day_number=4
3. 判断阶段：冷启动期（第1-15天）
4. 生成任务：优化主图、发布视频、联系达人
5. 输出报告

**输出**：
```markdown
# 🎯 今日任务清单（第4天）

## 当前状态
- 商品数：20
- 订单数：0
- 阶段：冷启动期

## 今日最重要的3件事

### 任务1：优化商品主图
...
```

## 相关参数

参考 `config/parameters.yaml` 中的：
- `cold_start_budget` - 冷启动预算分配
- `short_video` - 短视频制作参数
