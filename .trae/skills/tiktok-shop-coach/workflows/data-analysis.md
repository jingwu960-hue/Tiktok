# 数据分析工作流程

## 触发条件

用户询问"数据分析"、"曝光"、"点击"、"转化"、"GMV"等数据分析相关问题时触发。

## 执行步骤

### 1. 接收数据
从用户输入或 `shop-metrics.json` 获取以下数据：
- 曝光量（impressions）
- 点击量（clicks）
- 加购数（add_to_cart）
- 订单数（orders）
- GMV（gmv）

### 2. 计算核心指标
```
CTR = 点击 / 曝光 × 100%
转化率 = 订单 / 点击 × 100%
加购率 = 加购 / 点击 × 100%
客单价 = GMV / 订单
```

### 3. 对比行业基准
参考行业平均值（东南亚电商）：
- CTR：2-3%
- 转化率：1-2%
- 加购率：5-10%

### 4. 识别问题环节
根据指标判断问题所在：

| 指标 | 正常范围 | 问题判断 | 优化方向 |
|------|----------|----------|----------|
| CTR | 2-3% | <2% | 主图/标题问题 |
| 转化率 | 1-2% | <1% | 价格/详情页问题 |
| 加购率 | 5-10% | <5% | 产品吸引力问题 |

### 5. 根因分析
针对问题环节，分析可能原因：

**CTR偏低**：
- 主图不吸引眼球
- 标题关键词不精准
- 价格显示不友好

**转化率偏低**：
- 详情页信息不足
- 价格高于竞品
- 评价/销量基础差
- 配送时间过长

**加购率偏低**：
- 产品缺乏差异化
- 卖点不清晰
- 场景展示不足

### 6. 输出优化建议
针对每个问题，给出具体可执行的优化步骤。

### 7. 预期改善
根据优化措施，预估改善效果：
- CTR提升：+0.5-1%
- 转化率提升：+0.3-0.5%
- 订单增长：+50-100%

## 错误处理

### 场景1：数据缺失
**处理方式**：
1. 询问用户补充缺失数据
2. 如无法提供，使用 `shop-metrics.json` 中的历史数据
3. 明确标注数据来源

### 场景2：数据异常（如CTR>100%）
**处理方式**：
1. 检查数据计算是否正确
2. 询问用户确认原始数据
3. 排除异常值后重新计算

### 场景3：所有指标都偏低
**处理方式**：
1. 判断为冷启动期正常现象
2. 建议优先优化CTR
3. 提供冷启动期策略参考

## 输出格式

使用 `config/templates/data-report.md` 模板输出。

## 示例

**用户输入**："今天曝光5000，点击150，订单2，帮我分析一下"

**执行流程**：
1. 接收数据：曝光5000，点击150，订单2
2. 计算指标：CTR=3%，转化率=1.3%
3. 对比基准：CTR正常，转化率偏低
4. 识别问题：转化率偏低
5. 根因分析：详情页/价格问题
6. 输出建议

**输出**：
```markdown
# 📊 数据分析报告

## 今日数据
- 曝光：5,000
- 点击：150
- CTR：3%
- 转化率：1.3%
- 订单：2
- GMV：90,000 VND

## 问题诊断
- **最大问题**：转化率偏低
- **转化率**：1.3%（行业平均1.5%）

## 优化方案
### 优先级1：优化商品详情页
- 增加产品细节图
- 添加使用场景说明
- 优化越南语文案

### 优先级2：调整定价策略
- 检查竞品价格
- 考虑设置优惠券
- 测试不同价格点

## 预期改善
- CTR：3% → 3.5%
- 订单：2 → 3-4
```

## 相关参数

参考 `config/parameters.yaml` 中的：
- `platform_metrics` - 平台考核指标
- `short_video` - 短视频参数（如适用）
